Text colour classes

Use {{> main-palette}} on a span element to change the text colour. Some components have automated text colours based on the background colour (e.g. spotlight feature module). Use the .important span colour class to override the default text colour.

SASS usage and HTML colour class names

For SASS usage, the various web shades are available in two different palette maps: $basic-palette and $main-palette. The colour key names can be used in HTML class names on various components. For simplicity and a slightly slimmer build file, certain components only allow colors from $basic-palette (such as banner), while others allow all colours from the $main-palette (such as span text colour or bold callout).

The $basic-palette map contains the university's official brand primary palette and web shades at full strength.


			$basic-palette: (
				lightblue: $lightblue,
				darkblue: $darkblue,
				blue: $blue,
				mediumblue: $blue,
				orange: $orange,
				accent: $orange,
				grey: $bluegrey,
			);

The $main-palette map contains the colours from the basic palette, shade variations, as well as UI element colours.


			$main-palette: (
				lightblue: $lightblue,
				darkblue: $darkblue,
				blue: $blue,
				mediumblue: $blue,
				darkerblue: $darkerblue,
				spiritblue: $spiritblue,
				orange: $orange,
				accent: $orange,
				grey: $bluegrey,
				lightgrey: $lightgrey,
				mediumgrey: $mediumgrey,
				coolgrey: $coolgrey,
				warmgrey: $warmgrey,
				bluegrey: $bluegrey,
				white: $white,
				emergency: $emergency,
				alert: $alert,
				success: $success,
				warning: $warning,
			);

There is also the $foundation-palette map, which contains all of the above colours, but also includes legacy variable/class names. In SASS usage, this palette should typically not be used unless legacy support is required for the component. The bgcolor utility class uses this palette. See also: legacy colour support.

Support for Global v3.0 legacy colours

Global 3.0 styles included support for several v2.1 legacy class names, as well as 10 different shades of the three grey brand colours.

In v3.1, v2.1 legacy class names are no longer supported (e.g. "pink", "green", "yellow").

In v3.1, the extensive 10-shade variations have been removed (e.g. "coolgrey10", "coolgrey20", "warmgrey30", etc). However, all former HTML colour class names have legacy support in v3.1 and will be rendered as a one of the light/medium/full strength versions in the web palette. Where possible/as you come across them, the HTML class name should be updated to one of the fully supported light/medium/full strength shades instead.

cool/warm/bluegrey10 through 30 will render as the light version of the grey. E.g. A class name of .coolgrey20 will display the HEX colour code for .lightcoolgrey. A class name of .warmgrey30 will display the HEX colour code for .lightwarmgrey.

cool/warm/bluegrey40 through 70 will render as the medium version of the grey. E.g. A class name of .coolgrey40 will display the HEX colour code for .mediumcoolgrey. A class name of .bluegrey50 will display the HEX colour code for .mediumbluegrey.

cool/warm/bluegrey80 through 90 will render as the full strength version of the grey. E.g. A class name of .coolgrey80 will display the HEX colour code for .coolgrey. A class name of .warmgrey90 will display the HEX colour code for .warmgrey.
