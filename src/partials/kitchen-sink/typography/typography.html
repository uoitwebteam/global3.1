Heading styles on body text

Heading styles, including specific heading sizes, can be applied to body text or headings using either the heading tag as a class (e.g. class="h3") or the .heading-font class.

Feature text links

Add the class .feature-link to create a feature text link.

Feature link
SASS usage

Text and underline colors can be controlled via the feature-text-link() or feature-text-link-style() mixins.

			
				// Basic styling (text color) with default orange underline
				@include feature-text-link($color: $lightblue);
	
				// Full styling
				@include feature-text-link-style($color: $lightblue, $hover: $accent, $size: rem-calc(16))
Parameter Value Description
$color colour variable Text colour
$hover colour variable Underline and chevron hover colour
$size font-size Font size in pixels

Code

Code snippets and blocks use the PrismJS library for syntax highlighting. Code snippets/blocks inside <code> elements automatically have syntax highlighting applied. To define the code language for a snippet/block, use the language-xxxx class. See the PrismJS docs for supported languages, and the note below on adding the appropriate language plugin.

Inline code snippets:

<code class="language-css">p { color: red }<code>

Code blocks should be placed inside a <pre> element:

<pre><code class="language-css">p { color: red }</code></pre>
{{#unlesspage 'index'}}

If syntax highlighting isn't being applied for a specific language, you may need to edit the src/files/js/lib/prism.js file and import the language plugin. For example:

import 'prismjs/components/prism-scss'; {{/unlesspage}}
