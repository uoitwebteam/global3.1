Card grid

Common card grid implementation styles and elements are shown here. {{#unlesspage 'index'}}For each example, the required element/class name is listed and highlighted in the markup. Other displayed classes and elements are also listed.{{/unlesspage}}

{{#unlesspage 'index'}}

Base card grid markup

{{> card-grid-block emptycolor=true}}
{{/unlesspage}}

Background colour

Use {{> main-palette}} to change the card background colour.

{{#unlesspage 'index'}}

Note that the colour class is added to two elements.

Required classes/elements:
  • .card-grid-bgcolor
  • .block [colour-class] - controls the text colours (auto-chosen text colour based on contrast ratios)
  • .bgcolor [colour-class] - outputs the actual background colour
{{/unlesspage}}
{{> card-grid-block color='lightblue' color2='darkblue'}}

Link options

Links can be added as an inline button element, or the entire card can be a block-level link.

{{#unlesspage 'index'}} Required classes/elements:
  • <a class="button"> link element, or
  • Block-level card <a> link element
Additional card-grid classes/elements shown:
  • .card-grid-bgcolor
{{/unlesspage}}

Background image

Images must be the same dimensions.

{{#unlesspage 'index'}} Required card-grid classes/elements:
  • .card-grid-image
  • .image element (with CSS background-image property)
  • Note the omission of the .bgcolor element for a full opacity image.
Additional card-grid classes/elements shown:
  • .icon element
  • Block-level card <a> link element
{{/unlesspage}}
{{> card-grid-block icon=true blocklink=true bgimage=true}}
